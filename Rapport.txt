4)- 
	Le projet est decoupé en plusieurs modules, premierment on a le module Card.ml pour les carte, le module FArray.ml pour les tableau fonctionnels
	, Fifo.ml pour implementer les file, PArray pour l'impletation des tableau 
	classique, ensuite il y'a le module XpatRandom.ml qui contient la fonction shuffle qui permet de generer la permutation initiale via la graine,
	ensuite il y'a le module XpatSolver.ml ou se trouve le main du projet
	et ou les premieres initialisations sont faites tel que la permutation, la generation du premier etat..., ensuite il y'a les deux modules XpatSolverValidate et XpatSolverSearch le premier comme son nom l'indique permet de rechercher une solution d'une certaine configuration, et le 
	deuxieme permet de valider les fichier donner par l'utilisateur.
